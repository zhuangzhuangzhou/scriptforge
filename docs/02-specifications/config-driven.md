# 配置驱动规范（Skills / Agents / Pipelines）

版本：v1.0.0
更新日期：2026-02-05

## 目标
确保系统的行为不依赖硬编码，所有核心流程能力通过数据库配置完成，并可由前端动态修改。

---

## 一、配置驱动范围

以下内容必须配置化：
- Skill 定义（prompt、输入变量、输出模板）
- Agent 定义（角色、目标、校验规则）
- Pipeline 结构（阶段顺序、Skill/Agent 调用关系）
- 质检策略（PASS/FAIL、重试次数、修订路径）

以下内容不得写死：
- webtoon-skill 的具体 prompt
- breakdown-aligner / webtoon-aligner 的检查标准
- 拆解批次、剧本批次规则（必须作为配置）

---

## 二、配置原则

1. **流程顺序固定，执行内容可配置**
   - “类型确定 → 拆解 → 剧本”不可变
   - 每阶段调用的 Skill/Agent 可配置

2. **输出结构必须严格跟随模板**
   - 模板由 Skill 配置提供
   - 任何输出必须通过模板结构校验

3. **校验规则是 Agent，而不是写死逻辑**
   - 校验可更换、可升级
   - PASS/FAIL 由 Agent 输出决定

4. **配置即规则**
   - 前端修改配置后，执行结果应即时生效
   - 不需要重新发布后端服务

---

## 三、前端可编辑范围

前端需要提供可视化编辑能力：
- Skill 编辑器：prompt、变量、输出模板
- Agent 编辑器：角色、目标、校验标准
- Pipeline 编辑器：阶段顺序、技能链路、重试策略

---

## 四、配置变更的生效规则

- 变更保存后，新的执行立即使用最新配置
- 正在运行的执行任务保持原配置，不中断
- 所有配置变更必须有版本记录

---

## 五、最低配置要求（不可缺少）

- `webtoon-skill`
- `breakdown-aligner`
- `webtoon-aligner`
- 一个默认 Pipeline
- 每阶段至少一个 Skill
- 拆解与剧本阶段必须绑定 Validator
