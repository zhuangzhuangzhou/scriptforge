## 🎯 设计理念

### 核心原则

**"不假设内置 Prompt 是最优的"**

我们采用分层设计，平衡易用性和灵活性：

1. **普通用户** - 开箱即用
   - 使用系统内置的 Skills 和 Agents
   - 一键开始拆解，无需配置
   - 专注于生成好的剧本

2. **高级用户** - 可定制优化
   - 可以查看内置 Skills 的 Prompt
   - 可以复制并修改自己的版本
   - 可以创建自定义的 Agent 工作流

3. **管理员** - 系统维护
   - 可以管理系统内置的 Skills/Agents
   - 可以更新和优化内置配置

## 🎨 用户体验设计

### 普通用户流程

```
1. 上传小说 ✓
2. 点击"开始拆解" → 自动使用内置 breakdown_agent
3. 查看拆解结果
4. 点击"生成剧本"
5. 查看/编辑剧本
```

**特点**：
- 零配置
- 一键操作
- 专注于结果

### 高级用户流程

```
1. 访问 /admin/skills
2. 查看"系统内置"标签页
3. 找到想优化的 Skill（如 conflict_extraction）
4. 点击"复制"按钮
5. 修改 Prompt 模板
6. 保存并测试
7. 在拆解时选择使用自己的版本
```

**特点**：
- 可以优化 Prompt
- 可以创建自定义工作流
- 不影响系统内置配置

## 💡 最佳实践

### 优化 Prompt 的建议

1. **从复制开始**
   - 不要从零开始写
   - 复制内置的，然后微调

2. **小步迭代**
   - 每次只改一个地方
   - 立即测试效果

3. **保留示例**
   - 在 example_input/output 中记录好的案例
   - 方便后续测试

4. **记录改进**
   - 在 description 中说明为什么修改
   - 方便团队协作

### 创建 Agent 的建议

1. **保持简单**
   - 不要一次添加太多步骤
   - 先验证核心流程

2. **合理的错误处理**
   - 关键步骤：`on_fail: "stop"`
   - 可选步骤：`on_fail: "skip"`

3. **清晰的变量命名**
   - 使用有意义的 output_key
   - 方便后续引用

## 🎯 核心优势

### 1. 易用性

- **普通用户**：零配置，开箱即用
- **高级用户**：可以优化，但不强制
- **渐进式学习**：从使用到定制

### 2. 灵活性

- **不假设最优**：承认内置 Prompt 可能不完美
- **支持定制**：用户可以根据自己的需求优化
- **版本隔离**：用户的修改不影响系统内置

### 3. 可维护性

- **配置化**：不需要改代码
- **可测试**：UI 直接测试
- **可追溯**：每个 Skill 都有版本和历史

## ⚠️ 注意事项

### 数据库迁移

确保执行了所有迁移：

```bash
cd backend
alembic upgrade head
```

### 内置资源初始化

首次启动时会自动创建内置 Skills 和 Agents。如果需要重新初始化：

```python
# 在 Python shell 中
from app.core.init_simple_system import init_simple_system
from app.core.database import AsyncSessionLocal

async def reinit():
    async with AsyncSessionLocal() as db:
        await init_simple_system(db)

import asyncio
asyncio.run(reinit())
```

### 权限控制

当前所有用户都可以访问 Skills/Agents 管理。如需基于 tier 的权限控制，需要在前端添加检查。
