# 模型管理功能 - 快速开始指南

## 🚀 5分钟快速部署

### 步骤 1：生成加密密钥

```bash
# 生成32字节的加密密钥
python3 -c "import os, base64; print('ENCRYPTION_KEY=' + base64.b64encode(os.urandom(32)).decode())"
```

**将输出的密钥添加到环境变量：**
```bash
# 方式1：临时设置（当前终端有效）
export ENCRYPTION_KEY="<生成的密钥>"

# 方式2：永久设置（添加到 ~/.bashrc 或 ~/.zshrc）
echo 'export ENCRYPTION_KEY="<生成的密钥>"' >> ~/.bashrc
source ~/.bashrc
```

### 步骤 2：运行数据库迁移

```bash
cd backend

# 运行迁移
alembic upgrade head
```

**预期输出：**
```
INFO  [alembic.runtime.migration] Running upgrade -> eece6c8e3bad, add model management tables
```

### 步骤 3：初始化数据

```bash
# 初始化默认数据（2个提供商，5个模型，计费规则等）
python3 scripts/init_model_data.py
```

**预期输出：**
```
开始初始化模型管理数据...
✓ 创建提供商: OpenAI
✓ 创建提供商: Anthropic
✓ 创建模型: GPT-4 Turbo
✓ 创建模型: GPT-3.5 Turbo
...
✓ 数据初始化完成！
```

### 步骤 4：访问管理界面

1. 启动后端服务（如果还没启动）
2. 启动前端服务（如果还没启动）
3. 以管理员身份登录
4. 访问：`http://localhost:3000/admin/models`

## 📝 使用指南

### 1. 添加新的模型提供商

**场景**：您想使用阿里云的通义千问模型

**步骤：**
1. 进入"提供商管理"标签页
2. 点击"新建提供商"
3. 填写信息：
   - 提供商标识：`aliyun`
   - 显示名称：`阿里云`
   - 提供商类型：`openai_compatible`
   - API 端点：`https://dashscope.aliyuncs.com/compatible-mode/v1`
4. 点击"保存"

### 2. 配置模型

**场景**：添加通义千问模型

**步骤：**
1. 进入"模型配置"标签页
2. 点击"新建模型"
3. 填写信息：
   - 提供商：选择"阿里云"
   - 模型标识：`qwen-turbo`
   - 显示名称：`通义千问 Turbo`
   - 最大 Token 数：`8000`
   - 超时时间：`120` 秒
   - 默认温度：`0.7`
4. 点击"保存"

### 3. 添加 API 凭证

**场景**：添加阿里云的 API Key

**步骤：**
1. 进入"凭证管理"标签页
2. 点击"新建凭证"
3. 填写信息：
   - 提供商：选择"阿里云"
   - 凭证名称：`阿里云主账号`
   - API Key：`sk-xxxxxxxxxxxxx`
   - 配额限制：`1000000`（可选）
4. 点击"保存"

**安全提示：**
- API Key 将使用 AES-256-GCM 加密存储
- 传输过程使用 HTTPS 加密
- 界面上只显示脱敏后的 API Key

### 4. 设置计费规则

**场景**：为通义千问设置计费规则

**步骤：**
1. 进入"计费规则"标签页
2. 点击"新建规则"
3. 填写信息：
   - 模型：选择"通义千问 Turbo"
   - 输入 Token 价格：`0.5` 积分/1K tokens
   - 输出 Token 价格：`1.0` 积分/1K tokens
   - 最低消费：`0.1` 积分（可选）
   - 生效时间：立即生效
4. 点击"保存"

### 5. 测试凭证

**场景**：验证 API Key 是否有效

**步骤：**
1. 进入"凭证管理"标签页
2. 找到要测试的凭证
3. 点击"测试"按钮
4. 等待测试结果

**可能的结果：**
- ✅ 成功：凭证有效
- ❌ 失败：凭证无效或已过期

## 🔧 常见问题

### Q1: 如何切换默认模型？

**A:** 进入"模型配置"标签页，找到要设置为默认的模型，点击"设为默认"按钮。

### Q2: 如何查看 API Key？

**A:** 出于安全考虑，API Key 在界面上只显示脱敏后的版本（如 `sk-***...***abc`）。如果需要查看完整的 API Key，需要直接访问数据库（不推荐）。

### Q3: 如何更新 API Key？

**A:** 进入"凭证管理"标签页，点击"编辑"按钮，重新输入新的 API Key，点击"保存"。

### Q4: 计费规则何时生效？

**A:** 计费规则根据"生效时间"和"失效时间"自动生效。系统会自动选择当前时间范围内的规则。

### Q5: 如何禁用某个模型？

**A:** 进入"模型配置"标签页，找到要禁用的模型，点击状态开关即可。

### Q6: 数据库中没有配置会怎样？

**A:** 系统会自动降级到环境变量配置（`OPENAI_API_KEY`, `ANTHROPIC_API_KEY` 等），保证向后兼容。

## ⚠️ 注意事项

### 安全相关

1. **ENCRYPTION_KEY 必须设置**
   - 生产环境必须设置此环境变量
   - 密钥应该保密，不要提交到版本控制

2. **API Key 保护**
   - 不要在日志中输出完整的 API Key
   - 定期轮换 API Key
   - 监控异常访问

3. **权限控制**
   - 只有管理员可以访问模型管理功能
   - 定期审查管理员权限

### 性能相关

1. **缓存建议**
   - 建议使用 Redis 缓存模型配置
   - 缓存过期时间：5-10分钟

2. **数据库优化**
   - 定期清理过期的计费规则
   - 监控数据库查询性能

3. **并发控制**
   - 避免同时修改同一个配置
   - 使用乐观锁防止冲突
