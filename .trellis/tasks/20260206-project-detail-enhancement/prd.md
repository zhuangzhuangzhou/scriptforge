# 项目详情页体验优化与Bug修复

## 目标
优化上传体验，完善信息展示，修复文件大小显示Bug，并改进 Console 交互逻辑。

## 需求详情

### 1. 文件上传体验优化
- **需求**: 上传文件后，自动将小说名称设置为文件名（去除后缀）。
- **逻辑**:
  - 上传成功后，检查当前项目名称。
  - 如果项目名称为空或为默认值（"新建项目"等），直接更新为文件名。
  - 如果项目名称已有内容且与文件名不一致，弹出 `Modal.confirm` 询问用户是否更新项目名称。

### 2. 统计数据展示
- **需求**: 文件上传后显示相关统计数据。
- **分析**:
  - **文件大小**: 上传后应立即显示（目前为 0.00MB，需修复）。
  - **章节/字数**: 准确数据需在"智能拆分"后才能得出。
  - **实现**:
    - 修复后端上传接口，确保写入 `original_file_size`。
    - 前端上传成功后刷新项目信息，确保文件大小正确显示。
    - 拆分完成后，确保刷新显示准确的总章节数和字数。

### 3. 完善小说类型
- **需求**: 扩充小说类型选项。
- **列表**:
  - 悬疑/惊悚
  - 玄幻/修真
  - 都市/言情
  - 历史/架空
  - 网游/竞技
  - 科幻/末世
  - 灵异/恐怖
  - 脑洞/同人

### 4. 状态中文显示
- **需求**: 项目状态全局统一使用中文。
- **映射表**:
  - `draft`: 草稿
  - `uploaded`: 已上传
  - `ready`: 已拆分 (就绪)
  - `parsing`: 剧情分析中
  - `scripting`: 剧本生成中
  - `completed`: 已完成

### 5. Console 交互优化
- **需求**:
  - 右下角 System Console 默认 **关闭** (`false`)。
  - 点击 "开始智能拆分" 按钮时：
    1. 自动 **打开** Console。
    2. 添加一条 "正在请求拆分接口..." 的日志。
    3. 拆分成功后，显示 "拆分完成" 日志。

### 6. Bug 修复
- **问题**: 文件上传后显示 0.00MB。
- **原因**: 后端 `upload_novel_file` 接口未保存文件大小到数据库。
- **修复**: 修改后端代码，获取 `file.size` 或计算 `len(content)` 并保存。

## 技术方案

### Backend
- 修改 `backend/app/api/v1/projects.py`:
  - 在 `upload_novel_file` 中计算文件大小并赋值给 `project.original_file_size`。

### Frontend (`Workspace.tsx`)
- **State**: `showConsole` 默认值改为 `false`。
- **Upload Handler**:
  - 解析 `file.name`。
  - 比较 `formData.name`。
  - 调用 `projectApi.updateProject` 更新名称。
- **Render**:
  - 替换硬编码的 `Status` 显示逻辑，使用映射函数。
  - 更新小说类型 `<select>` 选项。
- **Split Handler**:
  - 调用 `setShowConsole(true)`。
  - 调用 `addLog` (需暴露或传递给 ConsoleLogger，或者简单使用本地 state `setLogs`) 模拟进度。

## 验收标准
- [ ] 上传文件后，数据库中 `original_file_size` 有值，前端显示正确大小。
- [ ] 上传文件后，项目名称能自动更新或提示更新。
- [ ] 小说类型下拉框包含所有新类型。
- [ ] 页面所有状态标签均为中文。
- [ ] Console 默认隐藏，点击拆分时自动展开并有日志滚动。
