# 小说原文页面对接与UI优化

## 目标
对接小说章节目录与正文接口，优化页面布局，使其与整体高科技风格保持一致。

## 需求详情

### 1. 接口对接 (Source Tab)
- **章节列表**：
  - 调用后端 API 获取章节目录（需确认后端是否有 `GET /projects/{id}/chapters` 接口，若无需新增）。
  - 替换左侧边栏的 `mockChapters` 数据。
  - 支持点击章节切换右侧正文内容。
- **正文内容**：
  - 根据选中的章节 ID，获取章节内容并显示。
  - 确保支持长文本滚动。

### 2. UI 布局优化
- **顶部标题栏**：
  - 在 `SOURCE` 标签页顶部增加一行小说标题栏。
  - 样式参考 `CONFIG` 页面的顶部样式（或 Dashboard 顶部），保持一致性（毛玻璃、渐变字）。
- **正文区域**：
  - 调整右侧小说正文容器，使其**填满父级高度和宽度**。
  - 去除不必要的内边距，让阅读体验更沉浸。

### 3. 后期调整 (已记录为后续子任务)
- 用户头像与用户名位置互换（用户名左，头像右）。
- 点击用户名弹窗样式全局统一（使用 `ProtocolModal` 风格）。

## 技术方案

### Backend
- 检查 `backend/app/api/v1/projects.py` 或 `chapters.py`。
- 确认是否存在 `GET /projects/{id}/chapters`。
- 确认返回数据包含 `id`, `title`, `content` (或单独获取内容以优化性能), `word_count`, `status`。

### Frontend (`Workspace.tsx`)
- **State**:
  - `chapters`: 存储章节列表。
  - `selectedChapter`: 存储当前选中章节详情。
- **Effect**:
  - 在切换到 `SOURCE` tab 时，若无数据则加载章节列表。
- **Render**:
  - 重构 `SOURCE` tab 的 JSX 结构。
  - 引入顶部 Header 组件（复用或新建）。
  - 调整 Flex 布局，确保 `flex-1` 正确作用于正文容器。

## 验收标准
- [ ] 左侧章节列表显示真实数据。
- [ ] 点击章节，右侧显示对应的真实正文内容。
- [ ] 页面顶部有与配置页一致的标题栏。
- [ ] 正文卡片填满右侧区域，无明显留白浪费。
