# System Console 功能修复

## Goal
修复和优化 System Console 组件的显示效果和用户体验。

## Requirements

### 0. 标题格式修改
- 当前格式: `第X轮/共Y轮 | 任务: XXX`
- 修改为: `任务: 拆解/质检 90% 第1次`
- 格式说明: `任务: {任务类型} {进度百分比} 第{轮次}次`

### 1. Raw 模式数据连贯性修复
- 问题: Raw 模式下数据被时间戳单独拆分，不连贯
- 参考: 后面的剧情拆解质量校验数据是连贯的
- 需要对比修改，确保 Raw 模式下数据也是连贯显示的

### 2. 任务完成后控制台卡住问题
- 问题: 任务完成后显示 "剧情拆解 Agent 完成"、"◈拆解完成，生成 21 个剧情点 (90%)" 后卡住不动
- 需要: 正确处理任务完成状态，重置相关状态

### 3. Formatted 按钮和内容优化
- 按钮文字: `Formatted` 改为 `Format`
- 内容颜色丰富化:
  - 去掉剧集前面的 ◆ 符号（如 `◆【第13集】` 改为 `【第13集】`）
  - 场景、角色、剧情、钩子 采用不同的样式颜色来表现

### 4. 输出文本优化
- 关键动作使用【】包裹
- 示例: `质量检查【通过】，第 1 轮完成。`
- 不同类型的【】内容给予不同颜色处理

### 5. 统一文字名称
- `网文改编剧情拆解` -> `剧集拆解`
- `剧情拆解质量校验` -> `质量检查`

## Acceptance Criteria

- [ ] 标题显示为新格式: `任务: 拆解/质检 90% 第1次`
- [ ] Raw 模式下数据连贯显示，不被时间戳拆分
- [ ] 任务完成后控制台正常结束，不卡住
- [ ] 按钮文字改为 `Format`
- [ ] 剧集标题去掉 ◆ 符号
- [ ] 场景/角色/剧情/钩子 有不同颜色区分
- [ ] 关键动作用【】包裹并有颜色区分
- [ ] 所有 `网文改编剧情拆解` 改为 `剧集拆解`
- [ ] 所有 `剧情拆解质量校验` 改为 `质量检查`

## Technical Notes

### 涉及文件
- `frontend/src/components/ConsoleLogger.tsx` - 主组件
- `frontend/src/hooks/useBreakdownLogs.ts` - 日志 WebSocket Hook
- `frontend/src/hooks/useConsoleLogger.ts` - 日志管理 Hook

### 颜色方案建议
- 场景: `text-blue-300`
- 角色: `text-amber-300`
- 剧情: `text-emerald-300`
- 钩子: `text-pink-300`
- 【通过】: `text-green-400`
- 【失败】: `text-red-400`
- 【警告】: `text-yellow-400`
