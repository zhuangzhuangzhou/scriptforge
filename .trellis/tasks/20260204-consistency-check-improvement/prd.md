# 一致性检查功能完善

## 目标
将一致性检查从基础的逻辑/人物校验扩展为全方位的剧本质量审计系统，确保改编后的剧本在时间线、场景和风格上保持高度连贯。

## 需求

### 1. 扩展检查维度
- **时间线一致性 (Timeline Consistency)**：检查故事发生的时间顺序是否冲突（如：前一秒是黑夜，下一秒变成了正午）。
- **场景连续性 (Scene Continuity)**：检查场景切换、人物位置、道具状态在连续场景中的合理性。
- **对话风格一致性 (Dialogue Style)**：确保同一个角色在不同场景中的口吻、习惯用语保持一致。

### 2. 深度分析与修复建议
- 每一项不一致点不仅要标出，还需要 AI 生成“修改建议”。
- 增加“一致性评分”机制（0-100分）。

### 3. 集成与持久化
- 完善 `ConsistencyCheck` 模型的使用，确保检查历史可追溯。
- 将检查结果集成到 `AITask` 的进度反馈中。

## 验收标准
- [ ] 实现时间线、场景连续性和对话风格的 AI 检查逻辑。
- [ ] 每次检查生成结构化的 JSON 结果，包含不一致描述、位置及修改建议。
- [ ] 检查结果正确保存至数据库，并可通过 API 查询。
