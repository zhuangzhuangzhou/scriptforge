# 批次显示问题系统分析

## 问题现象
前端显示"还在拆解批次 7",但实际上批次 9 正在执行

## 可能原因
1. **前端缓存**: `selectedBatch` 状态没有更新
2. **轮询延迟**: 30秒轮询间隔,状态更新不及时
3. **WebSocket 连接**: 连接的是旧任务 ID,没有切换到新批次
4. **批次列表未刷新**: `batches` 数组中的状态没有更新

## 需要检查的关键点

### 1. 批次状态更新流程
- `fetchBatches()` 何时被调用?
- `pollBatchProgress()` 如何更新 `selectedBatch`?
- WebSocket 消息如何触发批次切换?

### 2. 状态同步机制
- `selectedBatch` vs `batches` 数组中的状态
- `breakdownTaskId` 如何与批次关联?
- 批次完成后如何触发下一个批次?

### 3. 显示逻辑
- 哪里显示"还在拆解批次 X"?
- 显示的数据来源是什么?
- 是否有多个地方显示批次状态?

## 解决方案设计

### 方案 1: 实时状态同步
- 减少轮询间隔(30秒 → 5秒)
- WebSocket 消息触发立即刷新
- 批次完成时立即切换到下一个

### 方案 2: 状态管理优化
- 统一状态来源(单一数据源)
- 批次切换时强制刷新
- 添加状态同步日志

### 方案 3: UI 反馈增强
- 显示实时进度(批次 X/总数)
- 批次切换时显示明确提示
- 添加"刷新"按钮手动同步
