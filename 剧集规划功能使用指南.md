# 剧集规划功能使用指南

## 功能概述

剧集规划功能是剧本拆解的增强功能，能够智能地将章节内容分配到不同剧集，并为每个剧集筛选相关的拆解数据（冲突、钩子、角色、场景、情感）。

## 如何使用

### 1. 启动服务

#### 启动后端服务
```bash
cd backend
source venv/bin/activate
uvicorn app.main:app --reload
```

#### 启动 Celery Worker
```bash
cd backend
source venv/bin/activate
celery -A app.core.celery_app worker --loglevel=info
```

#### 启动前端服务
```bash
cd frontend
npm run dev
```

### 2. 执行拆解任务

1. 进入 Workspace 页面
2. 选择一个批次
3. 点击"开始拆解"按钮
4. 观察拆解进度：
   - 10%: 加载章节数据
   - 20%: 提取冲突
   - 35%: 识别情节钩子
   - 50%: 分析角色
   - 65%: 识别场景
   - 80%: 提取情感
   - **85%: 规划剧集结构** ← 新增步骤
   - 90%: 保存拆解结果
   - 100%: 任务完成

### 3. 查看剧集规划结果

拆解完成后，右侧详情区域将显示：

#### 一致性评分卡片
显示剧情一致性评分（0-100分）

#### 剧集规划概览卡片
- 显示总剧集数
- 显示涵盖的章节范围

#### 剧集详情卡片（可折叠）
每个剧集卡片包含：
- **标题栏**（点击可展开/折叠）
  - 剧集编号和标题
  - 主要冲突描述
  - 章节范围
- **详情区域**（展开后显示）
  - 关键场景
  - 冲突点列表
  - 剧情钩子列表
  - 角色列表
  - 场景列表
  - 情感列表

### 4. 交互操作

- **展开/折叠剧集**：点击剧集卡片的标题栏
- **默认状态**：第一集默认展开，其他剧集折叠
- **流畅动画**：展开/折叠时有平滑的动画效果

## 兼容性说明

### 向后兼容
如果拆解结果中没有剧集规划数据（`episodes` 为空），系统会自动降级到传统视图：
- 显示一致性评分
- 显示核心冲突列表
- 显示剧情钩子列表

这确保了旧数据和新数据都能正常显示。

### 数据迁移
已有的拆解结果不会自动生成剧集规划，需要重新执行拆解任务才能获得剧集规划数据。

## 技术细节

### AI 模型要求
- 建议使用较强的模型（如 GPT-4）以获得更好的规划效果
- 模型需要支持流式输出
- 需要配置正确的模型 API 密钥

### 性能影响
- 剧集规划步骤会增加约 5-10% 的拆解时间
- 对于大量章节的批次，规划时间可能较长
- 建议合理控制每个批次的章节数量（建议 10-20 章）

### 数据结构
剧集规划结果存储在 `plot_breakdowns.episodes` 字段中，格式为 JSONB 数组。每个剧集包含：
- `episode_number`: 剧集编号
- `title`: 剧集标题
- `main_conflict`: 主要冲突
- `key_scenes`: 关键场景列表
- `chapter_range`: 章节范围 [起始章节, 结束章节]
- `conflicts`: 该集的冲突列表
- `plot_hooks`: 该集的钩子列表
- `characters`: 该集的角色列表
- `scenes`: 该集的场景列表
- `emotions`: 该集的情感列表

## 常见问题

### Q: 为什么没有显示剧集规划？
A: 可能的原因：
1. 拆解任务尚未完成
2. 拆解任务失败
3. 使用的是旧数据（需要重新拆解）
4. AI 模型配置错误

### Q: 如何调整剧集规划的质量？
A: 可以通过以下方式优化：
1. 使用更强的 AI 模型
2. 调整提示词（在 `breakdown_tasks.py` 中）
3. 确保章节内容质量良好
4. 合理控制批次的章节数量

### Q: 可以手动编辑剧集规划吗？
A: 当前版本不支持手动编辑，这是后续优化的方向。

### Q: 剧集规划会影响剧本生成吗？
A: 当前版本的剧集规划主要用于展示和分析，暂不直接影响剧本生成。未来版本可能会将剧集规划结果用于指导剧本生成。
