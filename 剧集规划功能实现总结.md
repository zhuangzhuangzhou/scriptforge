#### 1.3 拆解任务流程调整 ✅
**文件**: `backend/app/tasks/breakdown_tasks.py`

新增步骤：
- 在提取情感（80%）之后，保存结果（90%）之前
- 新增"规划剧集结构"步骤（85%）
- 调用 `_plan_episodes_sync()` 函数

新增函数：
```python
def _plan_episodes_sync(
    conflicts: list,
    plot_hooks: list,
    characters: list,
    scenes: list,
    emotions: list,
    chapters: list,
    model_adapter,
    task_config: dict,
    log_publisher=None,
    task_id: str = None
) -> list
```

功能：
- 基于拆解结果，AI智能规划剧集结构
- 将章节内容分配到不同剧集
- 为每个剧集分配对应的冲突、钩子、角色、场景、情感
- 支持流式输出

### 3. 前端组件重构 ✅

#### 3.1 BreakdownDetail 组件重构
**文件**: `frontend/src/pages/user/Workspace/PlotTab/BreakdownDetail.tsx`

**重大变更**：从"按类型展示"改为"按剧集展示"

**新增组件**：
- `EpisodeCard`: 剧集详情卡片（可折叠）

**展示结构**：
```
- 一致性评分卡片
- 剧集规划概览卡片（显示共X集）
- 第一集详情卡片（默认展开）
  - 关键场景
  - 冲突点
  - 剧情钩子
  - 角色
  - 场景
  - 情感
- 第二集详情卡片（默认折叠）
- ...
```

**兼容性处理**：
- 如果 `episodes` 为空，则显示传统视图（核心冲突 + 剧情钩子）
- 保证向后兼容

**交互功能**：
- 点击剧集卡片标题栏可展开/折叠
- 默认展开第一集
- 使用 Framer Motion 实现流畅的折叠动画

### 9. 注意事项

1. **AI 模型依赖**：
   - 剧集规划功能依赖 AI 模型
   - 需要确保模型配置正确
   - 建议使用较强的模型（如 GPT-4）以获得更好的规划效果

2. **性能考虑**：
   - 剧集规划会增加拆解任务的执行时间（约 5-10%）
   - 对于大量章节的批次，规划时间可能较长

3. **数据一致性**：
   - 确保 AI 正确筛选每个剧集的数据
   - 建议在提示词中明确章节范围的判断逻辑
